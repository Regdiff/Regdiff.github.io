论文地址：CAKE: Cascading and Adaptive KV Cache Eviction with Layer Preferences

目前做 KV cache 个人理解有两个门派，一个是从算法层面由 H2O，StreamingLLM 领航到 SnapKV, PyramidKV，主要聚焦于怎么选这些 token (importance metric) 从而维持 accuracy；另一个是从硬件层面有 Minference 引航的有哪些 pattern 是利于 kernel 编写并且跑的很快的。本文属于前者，based on SnapKV, PyramidKV, D2O，创新在于把所有环节变的动态，随着 layer-ID 动，随着 prefill stage 动。

作者认为，attention map 具有层不均匀性 （layer-ID 动），一个最优的方案是给每层动态的分配不同的 buffer size。如何分配 buffer size？需要提出一个合理的指标。如图 1, spatial 维度选择类似 attention sink 的 metric，因为有些层可能存在 sink 但是有些层不存在，temporal 维度选择类似 D2O 的 variance metric，因为随着 prefill stage 的进行 (prefill stage 动），会呈现类似次对角线的 shift。综合两者调参得到了图 2 的 buffer size 分配流程。在图 3 中有一个实现上的细节，就是前面几个 layer 可以先保留多一点 token，等到后面的 layer 不够再删掉，一个类似缓存 buffer 的设计。

作者认为，SnapKV 的 metric 还能改进，图 4 中还加了一个 variance 项。我觉得是有道理的，variance 也能体现不均匀性 / 动态，这里可以看成随着 prefill stage 动。
